STEP 2 — CREATE A DESIGN SPEC (NOVAHEALTH-STYLE, ASCULTICOR CONTENT)
Create a concrete design system and component spec that matches the reference style:

A) VISUAL LANGUAGE
- Background: deep charcoal/near-black with subtle vignette; optional starfield/particle dust.
- Surfaces: glassmorphism cards (semi-transparent, blur, thin border, soft inner highlight).
- Radius: large (16–24px) consistent across cards/buttons.
- Shadow: soft, layered; avoid harsh outlines.
- Typography: modern sans (use existing font if present; otherwise pick one web-safe or already in deps). Use clear hierarchy: page title, section titles, card metric, supporting label.
- Accents: limited neon palette (teal/cyan + violet + amber for warnings; red for critical). Use sparingly for icons, strokes, small gradients, and glow.
- Micro-animations: subtle hover lift, glow intensify, chart shimmer, nav pill slide, particle drift. Respect prefers-reduced-motion.

B) LAYOUT PATTERN (MATCH REFERENCE)
- Top: pill-style navigation bar (even if sidebar remains, add a top contextual bar on desktop; on mobile collapse sidebar into drawer).
- Center stage: a “Hero Visualization Panel” on the Dashboard—3D/animated anatomical visualization (HEART-themed to match AscultiCor; skeleton is acceptable only as a fallback).
- Left column: patient/session identity and vitals/quick stats panels (adapt to AscultiCor: “Monitoring Status”, “Connected Devices”, “Recent Alerts”).
- Right column: activity, recent events, AI analytics panels (adapt to AscultiCor predictions & latency & offline devices).
- Use a 12-column grid with consistent gutters and vertical rhythm.

C) COMPONENTS TO BUILD (REUSABLE)
Implement reusable UI primitives:
- AppShell (sidebar + optional top pill bar + content container)
- GlassCard (base surface)
- MetricCard (big number + sparkline + status badge)
- StatusBadge (OK / Needs attention / Critical)

- PillTabs (top navigation style)
- DataList (Recent Sessions / Alerts list with compact rows)
- SectionHeader (title + right action)
- ChartContainer (consistent styling for charts)
- Skeleton loading states
- Empty states (no data)
